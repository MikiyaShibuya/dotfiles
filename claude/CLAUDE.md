# Global Code Review Checklist

## std::function
- 呼び出し前にnullチェック (`if (func)`) があるか
- 引数は値渡し+ムーブを検討（コピーコストを避ける）
- キャプチャのライフタイムは安全か

## シグネチャ変更時
- 引数の渡し方（値/const参照/右辺値参照）は適切か
- ムーブセマンティクスを活用できるか
- 既存の呼び出し元への影響

## スマートポインタ
- 所有権は明確か
- `shared_ptr` vs `unique_ptr` の選択は適切か
- 循環参照の可能性はないか

## 一般
- 防御的プログラミング: 単体で正しく動作するか（現在の呼び出し元だけでなく）
- エラーハンドリング: 失敗ケースは適切に処理されているか

# Git Safety Rules (CRITICAL)

## 禁止コマンド
以下のコマンドは**絶対に実行禁止**:
- `git reset --hard` - データ消失のリスク
- `git push --force` (main/masterブランチ)
- `git clean -fd` - untrackedファイル消失のリスク

## 必須プラクティス
1. **gitコマンド実行前にディレクトリ確認**
   - 必ず `cd /absolute/path && git ...` の形式で明示的に指定
   - サブモジュールと親リポジトリを混同しない

2. **間違ったコミットの修正方法**
   - `git revert` を使用（履歴を保持）
   - または、ユーザーに状況を説明して対処方法を相談

3. **破壊的操作の前はユーザーに確認**
   - reset, force push, rebase, clean など

## コミット作成ルール
1. **適度な粒度でコミットを分割**
   - 一度に大きな変更をコミットせず、論理的な単位で複数のコミットに分ける
   - 各コミットは単一の目的・変更を表すようにする

2. **amendの確認**
   - 最近作られたコミットを微修正する場合、amendするかどうかをユーザーに対話的に確認する
   - 勝手にamendしない

3. **コード・関連ファイルの同期**
   - 機能追加/削除時は、その機能に言及・依存する全てのファイルを同一コミットで更新する
   - 対象: ドキュメント（README, TODO, SPEC, CHANGELOG）、テスト（コード, 仕様書）、
     ビルド設定、CI/CD、サブモジュール
   - 原則: 1つのコミットでcheckoutすれば全てが整合する状態を維持する
   - コミット前チェック: 変更した関数/クラス名でgrepし、更新漏れがないか確認する

4. **untrackedファイルの確認**
   - 不要なuntrackedファイルをコミットしないように注意する

## 探索で見つからない場合は範囲を広げるか確認する (2026-01-20)
**問題**: センサー位置の定義場所を調査する際、限定的な検索で見つからなかったにも関わらず「見つからなかった」と結論づけ、`config/robot.py`を見落とした

**再発防止**:
- 何かを探して見つからない場合、以下のいずれかを実施する:
  1. **探索範囲を広げて再検索**: ディレクトリ範囲を広げる、別のキーワードで検索する
  2. **探した範囲と探し方をユーザーに提示**: 「〇〇の範囲で△△を検索しましたが見つかりませんでした。より広く探しますか？」と確認する
- **「見つからなかった」と結論づけない**: 一部を探して見つからなかっただけで「存在しない」と断言しない
